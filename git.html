Introdução ao Git e GitHub

O que é Git?

Git é um sistema de controle de versão popular. Foi criado por Linus Torvalds em 2005 e é mantido por Junio ​​Hamano desde então.

É usado para:

  • Alterações no código de rastreamento
  • Rastreando quem fez alterações
  • Colaboração de codificação

O que o Git faz?

  • Gerencie projetos com repositórios
  • Clone um projeto para trabalhar em uma cópia local
  • Controle e rastreie alterações com preparação e confirmação
  • Branch and Merge para permitir o trabalho em diferentes partes e versões de um projeto
  • Extraia a versão mais recente do projeto para uma cópia local
  • Enviar atualizações locais para o projeto principal

Trabalhando com Git

  • Inicialize o Git em uma pasta, tornando-o um Repositório
  • O Git agora cria uma pasta oculta para acompanhar as alterações nessa pasta
  • Quando um arquivo é alterado, adicionado ou excluído, ele é considerado modificado
  • Você seleciona os arquivos modificados que deseja preparar
  • Os arquivos preparados são confirmados, o que solicita que o Git armazene um instantâneo permanente dos arquivos
  • O Git permite que você veja o histórico completo de cada commit.
  • O Git não armazena uma cópia separada de cada arquivo em cada commit, mas monitora as alterações feitas em cada commit!

Por que Git?

  • Mais de 70% dos desenvolvedores usam Git!
  • Os desenvolvedores podem trabalhar juntos em qualquer lugar do mundo.
  • Os desenvolvedores podem ver o histórico completo do projeto.
  • Os desenvolvedores podem reverter para versões anteriores de um projeto.

O que é GitHub?

  • Git não é o mesmo que GitHub.
  • GitHub fabrica ferramentas que usam Git.
  • GitHub é o maior host de código-fonte do mundo e pertence à Microsoft desde 2018.
  • Neste tutorial, focaremos no uso do Git com GitHub.
